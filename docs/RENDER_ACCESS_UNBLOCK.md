## Manual Render Blueprint Import

Perform these steps only after explicit deployment approval and after the
approved branch has been pushed to GitHub:

1. Open the Render dashboard.
2. Choose **New +**.
3. Choose **Blueprint**.
4. Connect or select the GitHub account or organization that owns the
   Unibridge repository.
5. Select the repository containing `render.yaml`.
6. Select the approved branch, currently expected to be `main` unless the user
   chooses a different branch.
7. Confirm Render detects `render.yaml` at the repository root.
8. Review the generated resources before creating them:
   - Web service: `campusconnect-api`
   - PostgreSQL database: `campusconnect-postgres`
9. When prompted for `UNIVERSITY_PORTAL_CORS_ORIGINS`, leave it blank for
   mobile-only preview or enter only exact trusted HTTPS browser origins.
10. Confirm `UNIVERSITY_PORTAL_DATABASE_URL` is sourced from
    `campusconnect-postgres`.
11. Confirm `UNIVERSITY_PORTAL_ENVIRONMENT` is `production`.
12. Confirm `UNIVERSITY_PORTAL_SECRET_KEY` is generated by Render or otherwise
    set securely inside Render.
13. Create the Blueprint.
14. Wait for database provisioning, Docker build, `alembic upgrade head`, and
    API deploy completion.
15. Copy the final public HTTPS service URL from the Render service overview.

Do not run EAS preview builds after Render deployment. EAS remains separately
blocked until hosted API smoke tests pass and the coordinator explicitly
approves EAS builds.

permissions such as Render Shell access, but those are outside this unblock
